AGISCI COME UN SENIOR FULL STACK DEVELOPER SPECIALIZZATO IN HTML5, CSS3 E JAVASCRIPT VANILLA ES6.

Devi progettare e generare un sistema completo per un ristorante a tema SATISFACTORY e FICSIT composto da:

1. AREA CLIENTE
2. AREA ADMIN
3. LAYER DATI CONDIVISO

L'obiettivo è che tutte le operazioni effettuate nell'area cliente siano immediatamente visibili nell'area admin.

---

## ARCHITETTURA OBBLIGATORIA

L'applicazione deve essere organizzata in modo modulare.

Creare una cartella:

/data

contenente:

=== storage.js ===

Questo file deve gestire l'intera persistenza dati dell'applicazione.

Utilizzare LocalStorage come database simulato.

Implementare funzioni centralizzate:

* getMenu()
* saveMenu()
* getReservations()
* saveReservations()
* getOrders()
* saveOrders()
* getReviews()
* saveReviews()
* getDashboardStats()
* initializeDatabase()

Tutte le pagine CLIENT e ADMIN devono utilizzare ESCLUSIVAMENTE queste funzioni.

Nessuna pagina deve accedere direttamente al LocalStorage.

---

## SINCRONIZZAZIONE DATI

Quando un utente:

* effettua una prenotazione
* invia una recensione
* crea un ordine

i dati devono essere salvati nello storage condiviso.

L'area ADMIN deve leggere gli stessi dati.

Ogni modifica effettuata dall'admin deve aggiornare immediatamente il database simulato.

Utilizzare obbligatoriamente:

window.addEventListener("storage")

per sincronizzare automaticamente i dati tra più tab aperte.

Implementare inoltre un sistema di refresh automatico delle viste senza necessità di ricaricare la pagina.

# PRENOTAZIONE TAVOLI INTERATTIVA 3D AVANZATA

La pagina `prenotazione.html` deve includere una planimetria tridimensionale interattiva del ristorante ispirata ai sistemi di gestione industriale FICSIT presenti nell'universo SATISFACTORY.

## IMPORTANTE

NON utilizzare una semplice griglia 2D.

NON rappresentare i tavoli come semplici quadrati o pulsanti.

NON utilizzare select, checkbox o layout tabellari tradizionali.

La disposizione logica dei tavoli può essere 4x4 oppure 5x5, ma la visualizzazione deve essere renderizzata in pseudo-3D.
